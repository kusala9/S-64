THE RNC (HCRF) TEST DATA SET
============================

This Test Data Set has been produced for use when type approving ECDIS operating in RCDS mode (using data encoded in the HCRF Version 2.00 format) as specified in IEC 61174 Edition 3, Annex H.

The data consists of a set of Raster Navigational Charts (RNCs) from the United Kingdom Hydrographic Office's (UKHO) Admiralty Raster Chart Service (ARCS). The data may be used when type approving ECDIS systems operating in RCDS mode using either ARCS data or data from the Royal Australian Navy Hydrographic Service's Seafarer service, since both of these services use the same data format and encryption mechanism.

Because ARCS and Seafarer only use encrypted data within their services, the RNC data supplied with this file are also encrypted in a way that is compatible with both the ARCS and Seafarer services.

For ease of distribution, the test data is contained on a single CD. However, to make the data conform to the HCRF specification, it must be split up into separate CDs before use.  A total of ten (10) CDs are needed to hold the entire data set.  Details for creating these CDs are provided below.


TO MAKE INDIVIDUAL TEST DATA CDs
================================

The CD contains eleven (11) directories.  Ten (10) of these directories contain data that should be copied to individual CDs, while the eleventh directory contains a sample set of charts permits. The directories that contain the data to be copied to individual CDs are labelled as follows:

Note: The Rc1_cor and Wk33_cor directories are copies of the Rc1_old and Wk33_00 directories but with some of the data corrupted.

When copying these directories to individual CDs, copy the entire contents of the directory so that the README.CDV file (located immediately below the directory) is held in the new CD's Root directory.

The individual CDs need to have their Volume IDs set to the values in the table above.

Having done the above, the ten (10) new CDs should faithfully duplicate HCRF CDs and so will work with existing ARCS and Seafarer compatible equipment.


SAMPLE CHART PERMITS
====================

Because the test data set is encrypted, systems will need to have a suitable set of chart permits to allow the data to be used.  These are contained in the eleventh directory, the 'Permit' directory.

There are two (2) files in the permits directory, 'gb.lcn' (licence) and 'gb.ncp' (chart permits). These files should be copied to a floppy disk for use during tests.

The chart permits are made to operate with the generic 'A1' test manufacturer codes as supplied to all manufacturers of ARCS and Seafarer compatable equipment.  Consequently, all manufacturers should be able to use these permits for their own tests.

For the actual Type Approval Tests, manufacturers may wish to use their own manufacturer codes.  In these cases, suitable codes will need to be provided.  To obtain system specific chart permits please apply to:

USE OF THE DATA WITHIN THE TYPE APPROVAL TESTS
==============================================

The following paragraphs detail the use of the test data set by reference to the paragraphs in Annex H of IEC 61174 edition 3.

Where a test does not require specific RNC Test Data this is identified against the appropriate test with the reference "No specific test data".

All postions and bearings quoted below are formatted in degrees, minutes and decimals of minutes, e.g. the position 51 50.75N equates to 51 degrees 50.75 minutes North and the bearing 316 4.9188 equates to 316 degrees 4.9188 minutes.

ANNEX B - THE CHART HEADER FILE
===============================

Annex B has been included to fully explain where information relating to an RNC, e.g. Edition Date, Latest NM, etc. can be found and/or accessed. Further details can be obtained with reference to UKHO publication NP 760(HCRF Raster Format Version 2.00), Section 4.4.

Information regarding the displayed chart image can be obtained by accessing the header file (<ChartID>.HDR). The header file for a particular chart can be found by accessing the appropriate Area Disc (RC) and navigating the directory structure until the <ChartID>.hdr is reached. The example below displays the location of the chart 2675 header file in relation to the Rc1 root directory.

    Rc1--|
         |- Chartcat
         |- Misc
         |- Raschart-|
                     |- 026-|
                            |- 2675-|
                                    |-2675.hdr

Below is the Header file (<ChartID>.HDR) for chart 2675 as it would be displayed in Wordpad/Notepad with Word Wrap on:

   HDRGB2675    20000720199911110002.0003       
   200002773English Channel
   1070 GB  20012000450280805000000000000.00
   No special remarks for this chart.
   222000210006003  


Below is the same header file edited to highlight the principle records:

     Issuing             Edition Date
    Authority             (YYYYMMDD)
       |                      |
       |      File Issue Date | HCRF Version
       |         (YYYYMMDD)   |       |
       |           __|___  ___|__    _|_
       |          |      ||      |  |   |
   HDR GB 2675    20000720199911110002.0003       
   200002773English Channel 1070 GB  20012000450280805000000000000.00
   |_______| No special remarks for this chart.    
       |     222000210006003                       
       |
    Latest NM
   (YYYYNNNNN)

Where Y = Year
      M = Month
      D = Day
      N = Number

The "File Issue Date" contains the date on which the current
version of the file was published.

The "Edition date" field contains the date of publication of
the current edition of the chart, as printed on the face of
the original paper chart.

The "Issuing Authority" for ARCS charts is the UKHO, this is signified by the two character code "GB".
